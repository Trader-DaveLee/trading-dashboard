# Trading Desk Dashboard v6.7

## 이 도구는 무엇인가요?
Trading Desk Dashboard는 개인 트레이더가 **매매 전 계획, 보유 중 대응, 종료 후 복기, 반복 패턴 정리**를 하나의 흐름으로 관리할 수 있게 만든 로컬 중심 트레이딩 도구입니다.

핵심 목적은 다음 4가지입니다.
1. 진입 전에 계획을 먼저 적는다.
2. 리스크와 규모를 수치로 통제한다.
3. 보유 중 주요 대응만 기록한다.
4. 종료 후 패턴과 실수를 다시 다음 매매에 연결한다.

---

## 가장 추천하는 사용 흐름

### 1) Overview에서 시작
- Quick Launch로 필요한 외부 도구를 엽니다.
- 최근 성과, 캘린더, 포트폴리오 상태를 확인합니다.
- 오늘 어떤 흐름인지 먼저 파악합니다.

### 2) Journal → Pre-Trade Context
- 티커, 방향, Entry/Exit Setup을 입력합니다.
- 체크리스트를 확인합니다.
- 시장 컨텍스트와 진입 논리, 무효화 기준을 적습니다.
- 여기서 중요한 것은 “왜 지금 이 거래를 하려는가”를 남기는 것입니다.

### 3) Journal → Risk & Execution
- 현재가, 손절가, 목표가를 먼저 입력합니다.
- 매매용 시드, 리스크(%), 레버리지를 설정합니다.
- Price Map의 거리(%), bp, RR, 예상 순손익을 확인합니다.
- 실제 체결 entries와 planned/filled exits를 관리합니다.

### 4) Live Management
- 본절 이동, 부분청산, 추가진입, 계획 변경처럼 중요한 이벤트만 기록합니다.
- 모든 생각을 적기보다, 나중에 복기 가치가 있는 이벤트만 남기는 것이 좋습니다.

### 5) Post-Trade Assessment
- 종료 시간과 홀딩시간을 확인합니다.
- 최종 손익, 최종 R, 사용 리스크, 수수료, 계좌 영향을 봅니다.
- 태그, 실수, review를 남깁니다.
- 수익/손실보다 “계획 대비 실제 실행 차이”를 보는 것이 중요합니다.

### 6) Library와 Playbook 활용
- Library에서 같은 셋업, 종목, 실수 태그 기준으로 복기합니다.
- Playbook에는 다시 실행할 가치가 있는 거래만 남깁니다.

## 중점적으로 써야 하는 기능

### Pre-Trade Context
손절 계산보다 먼저, **왜 지금 진입하는지**와 **어디서 무효화되는지**를 정리하는 섹션입니다.

### Risk & Execution
이 섹션의 핵심은 수익 극대화보다 **리스크 통제**입니다.
- 현재가 / 손절가 / 목표가
- 리스크(%)
- 레버리지
- 실제 체결 entries
- planned / filled exits
를 기준으로 구조를 설계하세요.

### Live Management
본절 이동, 부분청산, 추가진입, 계획 변경 등 나중에 복기 가치가 높은 이벤트만 기록합니다.

### Post-Trade Assessment
최종 PnL과 R뿐 아니라,
- 사용 리스크 / 허용 리스크
- 누적 수수료
- 잔여 리스크 / BEP
- 계좌 영향
을 함께 보며 복기합니다.

## 실전에서 유용한 사용 팁

- 진입 후 기록보다 **진입 전 계획 기록**을 우선하세요.
- 리스크(%)를 먼저 정하고, 레버리지는 필요한 증거금과 포지션 사용률을 조절하는 용도로 보세요.
- 체크리스트가 불안하면 규모를 줄이거나 패스하는 것이 맞습니다.
- Playbook은 많이 쌓는 것보다 **정말 다시 쓸 거래만 남기는 것**이 좋습니다.
- 주기적으로 JSON 백업을 내보내세요.

---

## 주의할 점

- 이 도구는 **시장 데이터를 자동 수집하지 않습니다**. 입력하는 가격, 손절, 목표는 사용자가 책임지고 넣어야 합니다.
- 데이터는 브라우저 내 저장소(LocalStorage 캐시 + IndexedDB 영속 저장)에 함께 기록됩니다. 두 저장소는 저장 시점 버전과 시간 정보로 동기화되며, 더 최신 저장본이 우선됩니다. 브라우저 초기화 또는 저장소 삭제 전에 반드시 백업하세요.
- 차트 증거는 타임프레임별 링크 구조를 사용합니다. Month / Week / Day / 4H / 2H / 1H / 30m / 15m / 5m / 3m / 1m 중 하나를 고르고 링크를 저장해 관리하세요.
- 기본 셋업/태그/실수/프롬프트 템플릿이 포함되어 있으므로, 새 트레이드를 만들 때 선택 후 필요한 부분만 덧붙여 사용하는 방식이 가장 효율적입니다.
